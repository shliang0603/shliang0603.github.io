1 C++中基础知识

1.1 C++和C的一些关联

C++是对C的扩展，C++是完全兼容C的，在导入头文件的时候，一般是不带.h的，就是为了和C区别开来的。
C++是对C语言的加强（以C语言为基础），是更好的C语言，并且完全兼容C语言的特性

C++以C语言为基础面;相对象的支持;类型加强，函数加强，异常处理

1.1.1 命名空间

命名空间，当使用的时候，该头文件没有定义全局命名空间，必须使用namespace std;这样才能正确使用cout

如果不适用命名空间的时候，在调用标准库中的函数的时候，就要这样写std::cout<<std::hex<<3.14<<std<<endl;
所以最好的方法就是开头直接声明使用命名空间，这样就好像它们被声明为全局变量一样

可以自己定义命名空间

命名空间：

1）当使用的时候，该头文件没有定义全局命名空间，必须使用namespace std；这样才能正确使用cout。若不引入using namespace std ,需要这样做。std::cout。
2）C++标准为了和C区别开，也为了正确使用命名空间，规定头文件不使用后缀.h。
3）C++命名空间的定义： namespace name { … }
4）using namespace NameSpaceA;
5）namespce定义可嵌套。
6）当声明命名空间的名字的时候，在调用的时候可以不写命名空间的名字，也可以写，但是命名空间名字的后面要加上::作用域符号

1.1.2 register关键字增强

register 关键字的作用：请求编译器让变量a直接放在寄存器里面（不是内存），这样执行的速度就会很快

  • 在C中register修饰的变量不能够获取地址的，但是C++编译器用自己的优化方式，是可以获取变量的地址的，此时编译器发现程序中需要register变量的地址时，register对变量的声明就变得无效。

1.1.3 变量检测增强

在C中重复定义多个同名的全局变量是合法的，但是在C++中，不允许同时定义多个同名的全局变量。C语言中多个同名的全局变量最终会被链接到全局数据区的同一个地址空间上。

1.1.4 struct 类型的增强

C语言中struct定义了一组变量的集合，C编译器并不认为这是一种新的类型，C++中的struct是一个新类型的定义声明

1.1.5 C++中所有的变量和函数都必须要有类型

在C中可以没有定义变量或这函数的类型

在C语言中
int f( )；表示返回值为int，接受任意参数的函数
int f(void)；表示返回值为int的无参函数
在C++中int f( );和int f(void)具有相同的意义，都表示返回值为int的无参函数

C++ 更加强调类型，任意的程序元素都必须指明类型

1.1.6 C++ 中新增Bool类型关键字

C++中的布尔类型：

C++在C语言的基本类型系统之上增加了bool
C++中的bool可取的值只有true和false
理论上bool只占用一个字节，
如果多个bool变量定义在一起，可能会各占一个bit，这取决于编译器的实现

true代表真值，编译器内部用1来表示
false代表非真值，编译器内部用0来表示

bool类型只有true（非0）和false（0）两个值
C++编译器会在赋值时将非0值转换为true，0值转换为false

1.1.7 C++对三目运算符的增强

在C++中三目运算符返回的不是a对应的值10，而是变量本身，所以此时的三目运算符表达式是可以作为左值的，返回的是变量a，然后重新赋值成30

结论：

1）C语言返回变量的值 C++语言是返回变量本身
C语言中的三目运算符返回的是变量值，不能作为左值使用
C++中的三目运算符可直接返回变量本身，因此可以出现在程序的任何地方
2）注意：三目运算符可能返回的值中如果有一个是常量值，则不能作为左值使用
(a < b ? 1 : b )= 30;
3）C语言如何支持类似C++的特性呢？

1.1.8 C/C++中的const

const是用来定义一个常量，当是在C中用const定义的常量是伪常量（冒牌货），C++中的是真常量

在C中的伪常量

C\++中的常量是真常量，原理是C\++编译器引入一个符号变，这个符号变并不是内存地址，它是通过把const定义的常量的常量名以key的形式存到符号表中，把常量的值以value的形式存到符号表中

结论：

C语言中的const变量
C语言中const变量是只读变量，有自己的存储空间
C++中的const常量
可能分配存储空间,也可能不分配存储空间
当const常量为全局，并且需要在其它文件中使用，会分配存储空间
当使用&操作符，取const常量的地址时，会分配存储空间
当const int &a = 10; const修饰引用时，也会分配存储空间

const 和 #define的相同之处

C++中的const常量类似于宏定义
const int c = 5; ≈ #define c 5
C++中的const常量与宏定义不同
const常量是由编译器处理的，提供类型检查和作用域检查
宏定义由预处理器处理，单纯的文本替换

“作用域的检查”解释一下：就是#define 定义的宏常量，即使在函数内部定义，在全局也是可以用的，如果不想在某个函数定义的宏在其他函数中使用，可以在该函数的结尾声明#undef 或#undef 常量名 而const定义的常量，如果是定义在函数内部中，在其他函数中是不可以调用的。例如：

2.1 C++ 对C函数的扩展

2.1.1 inline 内联函数

1、内联函数的定义和调用：

内联函数定义：直接在函数定义的时候在函数类型前加上关键字inline 即可
内联函数调用：和普通函数调用一样

2、内联函数的作用：

  • C++编译器直接将内联函数体插入在函数调用的地方
  • 内联函数没有普通函数调用时的额外开销（压栈、跳转、返回）

3、使用注意

  • 现在的C++编译器能够进行编译优化，即使一些函数没有声明成inline函数，也可以被编译器进行内联编译
  • 内联函数的函数体不能过于庞大
  • 内联函数只是一种请求，编译器不一定允许这种请求，所以有些函数即使定义成内联函数，编译的时候还是和普通函数是一样进行编译

4、内联函数的限制

  • 函数体中不能存在任何形式的循环语句
  • 函数体重不能存在过多的条件判断语句
  • 函数体不能过于庞大
  • 函数内联声明必须在调用之前

所以内联和宏定义中要避免参数使用++这种操作

2.1.2 默认参数

1、C++中可以在函数声明时为参数提供一个默认值，当函数调用时没有指定这个参数的值，编译器会自动用默认值代替

2、默然参数使用的规则

  • 默认参数必须放在非默认参数的后面
  • 如果出现使用了默认参数，那么其后面的所有参数都必须是默认参数。例如 void myfun(int a, int b, int c = 2, int d) 这种就是错误的，必须是void myfun(int a, int b, int c = 2, int d = 3) 默认参数出现后后面都必须是默认。

3、函数占位参数
占位参数只有参数的类型声明，而没有参数名声明，一般函数体内无法使用占位参数

函数占位参数使用：在函数调用时，函数占位参数也必须传入实参，否则报错

2.2 函数的重载

函数的重载就是同时定义多个名字相同的函数，但函数的参数必须至少满足下面的一个条件才是重载函数：

  • 函数参数的个数不同
  • 函数参数的类型不同
  • 函数参数的顺序不同
